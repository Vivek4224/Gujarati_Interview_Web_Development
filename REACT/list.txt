### **1. React માં ઘટનાઓને કેવી રીતે સંભાળવું?**
Reactમાં ઘટનાઓને camelCase સિyntax નો ઉપયોગ કરીને સંભાળવામાં આવે છે. ઘટનાઓના હેન્ડલર્સ ફંક્શન તરીકે લખાતા હોય છે અને તેઓ આપોઆપ એક ઈવેન્ટ ઓબ્જેક્ટ પસાર કરે છે.

---

### **2. React માં ઇવેન્ટ હેન્ડલર્સમાં આર્ગ્યુમેન્ટ્સ કેવી રીતે મોકલશો?**
તમે આર્ગ્યુમેન્ટ્સ એરો ફંક્શન અથવા બાઇન્ડિંગથી મોકલી શકો છો.

---

### **3. React માં સિન્થેટિક અને નેટિવ ઇવેન્ટ્સ વચ્ચે શું તફાવત છે?**
સિન્થેટિક ઇવેન્ટ્સ એ નેટિવ ઇવેન્ટ્સ માટેનો ક્રોસ-બ્રાઉઝર રેપર છે, જે નોર્મલાઇઝ થયેલી કાર્યક્ષમતા પ્રદાન કરે છે.

---

### **4. React માં ફોર્મ ઇનપુટ ઘટનાઓને કેવી રીતે સંભાળવું?**
ફોર્મ ઇનપુટ ઘટનાઓ જેમ કે `onChange` એ કંટ્રોલ્ડ કોમ્પોનન્ટ્સ સાથે ઉપયોગ થાય છે, જ્યાં યુઝર ઇનપુટ પર સ્ટેટ અપડેટ થાય છે.

---

### **5. React માં `onSubmit` ઇવેન્ટ હેન્ડલર શું છે?**
`onSubmit` ફોર્મ સબમિશન્સને સંભાળે છે, અને `event.preventDefault()` ક્યારેક ડિફોલ્ટ ફોર્મ સબમિશન વર્તનને રોકવા માટે ઉપયોગ થાય છે.

---

### **6. React માં ઇવેન્ટ ડિલીગેશન શું છે?**
ઇવેન્ટ ડિલીગેશનમાં, પેરેન્ટ એલીમેન્ટ પર એક જ ઇવેન્ટ લિસ્ટનર જોડવામાં આવે છે, જે ચાઈલ્ડ એલીમેન્ટ્સ માટે ઘટનાઓને સંભાળે છે, જેના કારણે પ્રદર્શન સુધરે છે.

---

### **7. React માં ઇવેન્ટના ડિફોલ્ટ વર્તનને કેવી રીતે રોકવું?**
`event.preventDefault()` નો ઉપયોગ કરીને ડિફોલ્ટ વર્તન, જેમ કે ફોર્મ સબમિટ થવા થી રોકી શકાય છે.

---

### **8. React માં ઇવેન્ટના પ્રોપાગેશનને કેવી રીતે રોકવું?**
`event.stopPropagation()` નો ઉપયોગ કરીને ઇવેન્ટને પેરેન્ટ એલીમેન્ટ્સ તરફ બબલિંગ થવા થી રોકી શકાય છે.

---

### **9. React માં ઇવેન્ટ્સના સંદર્ભમાં કંટ્રોલ્ડ અને અનકંટ્રોલ્ડ કોમ્પોનન્ટ્સ શું છે?**
કંટ્રોલ્ડ કોમ્પોનન્ટ્સ React સ્ટેટનો ઉપયોગ કરીને ફોર્મ ડેટાને સંભાળે છે, જ્યારે અનકંટ્રોલ્ડ કોમ્પોનન્ટ્સ DOM પર આધાર રાખે છે અને refs નો ઉપયોગ કરે છે.

---

### **10. Class components માં ઇવેન્ટ હેન્ડલર્સ માટે `this` ને કેવી રીતે બાઇન્ડ કરશો?**
Class components માં, તમે `this` ને ઇવેન્ટ હેન્ડલર્સ સાથે either constructor માં અથવા એરો ફંક્શનનો ઉપયોગ કરીને બાઇન્ડ કરી શકો છો.
