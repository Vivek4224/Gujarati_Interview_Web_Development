API Questions

1. API શું છે?  
API (Application Programming Interface) એ વિવિધ સોફ્ટવેર સિસ્ટમ્સને એકબીજા સાથે વાતચીત કરવા માટેના માધ્યમ તરીકે કાર્ય કરે છે. આ એ method અને ડેટા ફોર્મેટ્સને પરિભાષિત કરે છે જે સિસ્‍ટમ્સ વચ્ચે માહિતી આપવાની પ્રક્રિયાને સુચારૂ બનાવે છે.


2. REST શું છે?
REST (Representational State Transfer) એ વેબ સર્વિસ બનાવવાની એક આર્કિટેક્ટચરલ શૈલી છે, જે સ્ટેટલેસ કોમ્યુનિકેશન, HTTP method (GET, POST, PUT, DELETE), અને રિસોર્સ આધારિત URLs પર આધારિત છે.


3. Web API શું છે?  
Web API એ એક API છે જે HTTP પ્રોટોકોલ્સનો ઉપયોગ કરીને વેબ પર ઉપલબ્ધ છે. આ એપ્લિકેશન્સને ઈન્ટરનેટ દ્વારા એકબીજા સાથે સંપર્ક કરવા અને સામાન્ય રીતે RESTful સર્વિસીસનો ઉપયોગ કરીને ઈન્ટરએક્ટ કરવા સક્ષમ બનાવે છે.



4. SOAP અને REST માં શું ફરક છે?
- SOAP (Simple Object Access Protocol) એ એક પ્રોટોકોલ છે જે કડક માનકોથી બાંધી શકાય છે અને સંપર્ક માટે વધુ ઓવરહેડની જરૂર હોય છે.  
- REST એ એક આર્કિટેક્ટચરલ શૈલી છે જે સરળ અને વધુ લચીલા સંચાર પર આધારિત છે, અને સામાન્ય રીતે CRUD ઓપરેશન્સ માટે HTTP methodનો ઉપયોગ કરે છે.

---

5. JSON શું છે? 
JSON (JavaScript Object Notation) એ લાઇટવેઇટ ડેટા ઇંટરચેન્જ ફોર્મેટ છે જે ડેટા ઑબ્જેક્ટ્સને માનવ વાંચી શકાય તેવી રીતે દર્શાવવાનો ઉપયોગ થાય છે.

JSON નું ઉદાહરણ:  
json  
{  
  "name": "John",  
  "age": 30  
}



6. ડેટા પ્રાપ્ત કરવા માટે HTTP પદ્ધતિ શું છે? 
`GET` પદ્ધતિ સર્વરથી ડેટા પ્રાપ્ત કરવા માટે ઉપયોગ થાય છે.



7. નવી ડેટા બનાવવા માટે HTTP પદ્ધતિ શું છે? 
`POST` પદ્ધતિ સર્વર પર નવી ડેટા બનાવવા માટે ઉપયોગ થાય છે.



8. પહેલાથી આવેલા ડેટાને અપડેટ કરવા માટે HTTP પદ્ધતિ શું છે?  
`PUT` પદ્ધતિ સર્વર પર હાજર ડેટાને અપડેટ કરવા માટે ઉપયોગ થાય છે.



9. ડેટા હટાવવા માટે HTTP પદ્ધતિ શું છે? 
`DELETE` પદ્ધતિ સર્વર પર ડેટા હટાવવા માટે ઉપયોગ થાય છે.



10. APIમાં એન્ડપોઈન્ટ શું છે?
એન્ડપોઈન્ટ એ APIમાં કોઈ વિશિષ્ટ URL અથવા URI છે જે સંસાધન (resource) દર્શાવે છે અને HTTP method દ્વારા પ્રવૃત્તિઓ કરવા માટે ઍક્સેસ કરી શકાય છે (ઉદાહરણ તરીકે, `GET /users`).



11. API કી શું છે? 
API કી એ એક અનોખો ઓળખકર્તા છે જે ક્લાયન્ટ એપ્લિકેશનને API ઍક્સેસ કરતી વખતે ઓથેન્ટિકેશન માટે ઉપયોગ થાય છે. આ ખાતરી કરે છે કે વિનંતી વિશ્વસનીય સ્ત્રોતમાંથી આવી રહી છે.


12. OAuth શું છે?  
OAuth એ ઓથેન્ટિકેશન માટે એક ખૂલ્લો ધોરણ છે જે ત્રીજી-પક્ષ એપ્લિકેશન્સને યુઝર ડેટા ઍક્સેસ કરવાની પરવાનગી આપે છે, એના ક્રેડેંશિયલ્સને બહાર પાડ્યા વિના. આ ટોકન દ્વારા સુરક્ષિત ઍક્સેસને સક્ષમ બનાવે છે.



13. CORS શું છે? 
CORS (ક્રોસ-ઓરીજિન રિસોર્સ શેરિંગ) એ બ્રાઉઝર્સમાં લાગુ કરાયેલ એક સુરક્ષા ફીચર છે જે નિયંત્રિત કરે છે કે કેવી રીતે વેબ સર્વર પરના સંસાધનોને એક અલગ ડોમેનથી વિનંતી કરી શકાય છે.



14. APIમાં રેટ લિમિટિંગ શું છે? 
રેત લિમિટિંગ એ એક વ્યૂહ છે જે APIs દ્વારા ઉપયોગકર્તાને સર્વર પર નિર્ધારિત સમયગાળામાં કેટલાંક વિનંતીઓ કરવાની મર્યાદા આપે છે, જેથી દુરુપયોગ અટકાવી શકાય.



15. APIમાં વર્ઝનિંગ શું છે?  
API વર્ઝનિંગ એ એક પ્રથા છે જેમાં APIમાં ફેરફારો કરવા માટે નવા વર્ઝનોને રજૂ કરવામાં આવે છે, જ્યારે પહેલાના વપરાશકર્તાઓ માટે પાછો અનુકૂળતા જાળવવામાં આવે છે. આ URL પાથ, હેડર્સ, અથવા ક્વેરી પેરામીટર્સનો ઉપયોગ કરીને કરવામાં આવી શકે છે.



16. API ગેટવે શું છે?  
API ગેટવે એ સર્વર છે જે API વિનંતીઓ માટે પ્રવેશ બિંદુ તરીકે કાર્ય કરે છે, અને તે ઓથેન્ટિકેશન, લોડ બેલેન્સિંગ, વિનંતી રાઉટિંગ, અને પ્રતિસાદ એગ્રિગેશન જેવી બાબતોને સંભાળે છે.



17. વેબસોકેટ શું છે?  
વેબસોકેટ એ પ્રોટોકોલ છે જે ક્લાયંટ અને સર્વર વચ્ચે એક એકમાત્ર, લાંબી અવધિની કનેક્શન પર સંપૂર્ણ-દ્વિ-દિશાની સંચારને સક્રિય કરે છે, જે રિયલ-ટાઇમ એપ્લિકેશન્સ માટે આદર્શ છે.



18. JSON વેબ ટોકન (JWT) શું છે? 
JWT એ એક સંકુચિત, URL-સુરક્ષિત રીતે બંને પક્ષો વચ્ચે દાવાઓ પ્રદર્શન કરવાનો રસ્તો છે. આનો ઉપયોગ સામાન્ય રીતે યુઝર ઓથેન્ટિકેશન અને ઓથોરાઇઝેશન માહિતી API કૉલ્સમાં સુરક્ષિત રીતે મોકલવા માટે થાય છે.


19. RESTful API શું છે?  
RESTful API એ REST સિદ્ધાંતોને અનુસરતું API છે, જે HTTP method, Stateless સંચાર, અને URL દ્વારા ઓળખાતા સ્રોતોનો ઉપયોગ કરે છે. સામાન્ય રીતે આ JSON ફોર્મેટમાં ડેટા પાછું આપે છે.



20. સિંક્રનસ અને અસિંક્રનસ API કૉલ્સમાં શું તફાવત છે? 
- સિંક્રનસ API કૉલ્સ તે કોડના અમલને પ્રતિસાદ પ્રાપ્ત થવા સુધી બ્લોક કરે છે.  
- અસિંક્રનસ API કૉલ્સ કોડના અમલને પ્રતિસાદની રાહ જોઈને ચાલુ રહેવાની મંજૂરી આપે છે.

21. ડેટાબેઝ શું છે? 
ડેટાબેઝ એ ગોઠવેલા ડેટાનું સંકલન છે જે સરળતાથી ઍક્સેસ, મેનેજ અને અપડેટ કરી શકાય છે. આ માટે સામાન્ય રીતે ડેટાબેઝ મેનેજમેન્ટ સિસ્ટમ (DBMS) નો ઉપયોગ થાય છે.



22. રિલેશનલ ડેટાબેઝ શું છે?
રિલેશનલ ડેટાબેઝ એ ટેબલ્સમાં ડેટા સંગ્રહ કરે છે, જેમાં row અને કૉલમ્સ હોય છે, અને ટેબલ્સ એકબીજાને ફોરેન કી દ્વારા સંબંધિત થઈ શકે છે. ઉદાહરણ તરીકે MySQL, PostgreSQL અને Oracle.



23. SQL શું છે?  
SQL (Structured Query Language) એ એક ભાષા છે જે રિલેશનલ ડેટાબેઝ સાથે સંચાર કરવા અને તેમને મેનેજ કરવા માટે ઉપયોગ થાય છે. આ ભાષા ડેટાને ક્વેરી, અપડેટ અને મેનેજ કરવા માટે ઉપયોગી છે.


24. પ્રાઇમરી કી શું છે? 
પ્રાઇમરી કી એ ટેબલમાં રેકોર્ડ માટે અનોખો ઓળખકર્તા છે. તે ખાતરી કરે છે કે તે ફીલ્ડ માટે બે રેકોર્ડમાં સમાન વેલ્યુ નથી.  



25. ફોરેન કી શું છે?
ફોરેન કી એ એક ટેબલમાં આવેલો ફીલ્ડ છે જે બીજાં ટેબલના પ્રાઇમરી કી ને રેફર કરે છે, આથી બંને ટેબલ વચ્ચે સંબંધ સ્થાપિત થાય છે.



26. ડેટાબેઝમાં નોર્મલાઇઝેશન શું છે?
નોર્મલાઇઝેશન એ ડેટાબેઝમાં ડેટાને ગોઠવવાની પ્રકિયા છે, જે રેડન્ડન્સી અને ડિપેન્ડન્સીને દૂર કરવા માટે મોટા ટેબલને નાના ટેબલોમાં વિભાજિત કરે છે.



27. ડેનોર્મલાઇઝેશન શું છે?
ડેનોર્મલાઇઝેશન એ ડેટાબેઝમાં રેડન્ડન્સી પ્રવેશ કરાવવાની પ્રકિયા છે, જે વાંચન પ્રદર્શનને શ્રેષ્ઠ બનાવવા માટે લખાણના પ્રદર્શન અને સ્ટોરેજ જગ્યા માટે ખર્ચ ઊભો કરે છે.



28. SQL માં JOIN શું છે?
JOIN એ SQL ઓપરેશન છે જે બે અથવા વધુ ટેબલમાંથી rowને જોડે છે જે સંબંધિત કોલમના આધારે જોડાય છે, સામાન્ય રીતે ફોરેન કી.

JOINના પ્રકારો:
- INNER JOIN: તે rowને રિટર્ન કરે છે જેમણે બંને ટેબલમાં મૅચિંગ વેલ્યુઝ હોવી જોઈએ.
- LEFT JOIN: તે ડાબા ટેબલની તમામ row અને જોડીવાળી rowને દક્ષિણ ટેબલમાંથી રિટર્ન કરે છે.
- RIGHT JOIN: તે જમણી ટેબલની તમામ row અને જોડીવાળી rowને ડાબા ટેબલમાંથી રિટર્ન કરે છે.
- FULL OUTER JOIN: તે તમામ row રિટર્ન કરે છે જ્યારે ડાબા અથવા જમણા ટેબલમાં મૅચ હોય.

29. ડેટાબેઝમાં ઇન્ડેક્સ શું છે?
ઇન્ડેક્સ એ એક ડેટાબેઝ ઑબ્જેક્ટ છે જે ટેબલમાંથી row ને ઝડપી મેળવાની પ્રક્રિયાને મજબૂત બનાવે છે, જે ક્વેરીઝ માટે ચોક્કસ કોલમ આધારિત ઝડપી લૂકઅપ પ્રદાન કરે છે.


30. ડેટાબેઝ મેનેજમેન્ટમાં ACID શું છે?
ACID એ ગુણધર્મોનો સેટ છે જે ગ્યારાંટી આપે છે કે ડેટાબેઝ ટ્રાન્ઝેક્શન્સ વિશ્વસનીય રીતે પ્રક્રિયાવિહિત થાય છે:
- Atomicity: ટ્રાન્ઝેક્શનને એક એકમ તરીકે માનવામાં આવે છે, તે સંપૂર્ણપણે પૂર્ણ થાય છે અથવા સંપૂર્ણપણે રોલબેક થાય છે.
- Consistency: ટ્રાન્ઝેક્શન ખાતરી કરે છે કે ડેટાબેઝ એક માન્ય સ્થિતિમાંથી બીજી માન્ય સ્થિતિમાં બદલાય છે.
- Isolation: ટ્રાન્ઝેક્શન્સ એકબીજાથી અલગ રહે છે, જે મકબુલ ન થાય તે રીતે એકબીજાને અવરોધે છે.
- Durability: એક વાર ટ્રાન્ઝેક્શન પ્રતિબદ્ધ થાય પછી, ફેરફાર પર્મનન્ટ રહે છે.
